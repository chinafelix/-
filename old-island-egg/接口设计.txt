1.最新一期的期刊 {
  不需传参
  期刊的第几期数（例如：8）
  期刊详细数据{
    标题，封面图，类型（电影、句子、书籍），内容 
  }
  点赞数
  本人是否已经点赞
}

2. 上一期 / 下一期 {
  根据当前期的期数， 加1 或者减一

  其余的和接口1 一样
}

3.书单列表 {
  书单的封面图、 点赞数
  本人是否已经点赞
}

4.搜索书籍 {
  关键字搜索， 查询列表数据{
    名字， id 
  }
}

5. 根据id查询期刊详细信息 {
  包含多种类型期刊；
  书籍名称、封面图、简介、出版信息、短评、点赞数 {
    短评、点赞数
  }
  本人是否已经点赞
}

6.获取token {
  如果有多种登陆方式的话， {
    type，
    微信的话需要微信的接口返回数据
  }
}

7. 验证token {
  传入token参数
}

8.评论{
  发布短评， 同名短评 +1
}

9. 我喜欢 {
  所有的喜欢列表 {
    封面、句子、点赞数
  }

  我喜欢的期刊总数
}

10 点赞 {
  对当前的期刊点赞 {
    id， type
  }
}

11 取消点赞 {
  类似点赞
}

12 期刊点赞情况 {
  可直接在相应接口写，也可以独立一个接口，还没想好那个合适
}
